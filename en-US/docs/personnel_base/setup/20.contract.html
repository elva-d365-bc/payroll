    Labour Contract

    It is possible to enter employment contracts or their changes to the system. Also you can attach templates and print an employment contract directly from the system.

    Creating labor contract

    Should issue a new employee card where the employee's status is Active (Fill automatically). If an employee is hired who have been worked in the company previously, then for this employee needs to be create a new employee card. In the employee card all the necessary fields should be filled, what is necessary for completing the labor contract, except the salary calculation, employment date and payroll components. The completion of the employee card is described in the chapter Employee Card. Information on labor contract what is required for salary calculation and personnel should be entered in previously defined fields in the Labour Contract List card. Labour Contracts and their corrections/appendixes will vary by sign – by Alteration field enabled and by entry in field Main Contract No.. Following information should be stored when entering the labour contract:

    • No. – Labour contract number From No. Series;
    • Posting Date;
    • Status – Created, In Approval, Completed (in order to ensure the procedure for preparing and approving the labour contract);
    • Employee No. – number from Employee list;
    • First Name;
    • Last Name;
    • Social Security No.;
    • Employment Date;
    • Planned Termination Date – if the date is specified. Only informative purpose;
    • Position Code – code from list Positions;
    • Position Description;
    • Department Code – code from list Departments;
    • Department Name;
    • Social Sign Code – mandatory when hiring a new employee;
    • Salary Type – Aggregated or normal working time
    • Loading;
    • Cycle Calendar – enable if employee's working time is organized as shift;
    • Calendar Code;
    • Calendar Name;
    • Cycle Serial Number;
    • Test Period – if defined;
    • Test Period End Date – calculates automatically if it is defined for the entire company;
    • Post Order with LC - The mark on when the posting of an employment contract automatically will post also the personnel order for the recruitment of the employee.

    Fields to be filled in the case of alteration of the labour contract. More about the contract's alteration is described in the next paragraph:

    • Alteration – enabled if it's alteration of the contract;
    • Main Contract No. – reference to a posted labour contract for which this alteration is intended. Usually it is a contract with what the employee has been recruited;
    • Alteration Type Code – a code from list Labour Contract Alteration Types, for example, increasing/decreasing salary, unification of the positions, change the position etc. For each type of alteration is possible to define it' own document template.
    • Alteration Description;
    • Employee Position No. – Current No of employee's position;
    • Primary Position – enable, if this position is primary;
    • Employee Position Description.

    Field Contract No. to Replace intended only in case of replacement of the main labour contract No. without making any changes to the employee's card. In case the field Contract No. to Replace is completed, then the system is looking for a document posting date for active position and active payroll components attached to this position on a date of posting. All labour contract card's fields are not editable, for the user does not attempt to correct anything. In case if Contract No. to Replace system automatically disables the field Post Order with LC.

    • Social Status Code – choose from the Social Statuses list;
    • Create Tax Components – enable if the Social Status Code was entered.

    If the user uses the function, that after the labour contract posting, also personnel order is posting automatically, and in addition in the setup is enabled Create Tax Components, then by posting labour contract where tab's Accounting fields are left blank and disabled, taxes will not calculate correctly and the tax calculation will be corrected in the employee's card. In the tab Signatures you can select two signatories from the company's officials list.

    In the tab Template can be inserted the document template from the template classifier. By default, the template comes from Personnel Setup, however, if a template is indicated in the position card, the system will take the specified.

    Tab Add.info – information about when and what have entered the data in this card.
