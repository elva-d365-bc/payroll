    Salary book import form EDS

    In order not to perform manually changes to employee reliefs every month, import of salary book data is available: LV Payroll Role Center - Tasks - Salary Book Import List.

    A new document is created where the following information shall be entered:

    • No. - is entered manually;
    • Registration No. – is completed from the file ;
    • Company Name - is completed from the file;
    • Type – you can choose between two options:
      • All - imports all information about relief, is entered from the file;
      • Changed - imports only changes. EDS report's option must be include all changes ( both non-taxable minimum and all other changes). Otherwise the system will give an error message.
    • Non-Taxable Minimum Comp. Code – should be selected from the list;
    • From Date – the start of the period, is completed from the file with the option to edit;
    • From Date In File – filled from the file;
    • To Date – the end of the period, filled from the file;
    • Date Created - filled from the file.
    1. step: press button Import, set up the path to the file. It is possible to specify from what date system should processing employees' salary components. In case if no conversion of previous months will be made, this date can left as it is.
    2. step: When importing the data, they must be processed. Start the function Process. After data processing, you can view both - an imported data and the proposed changes.

    The filter can be removed when you open the appropriate column and, if necessary, can manually make corrections (for example, the amounts of the non-taxable minimum).

    The non-taxable minimum amounts can only be corrected in the column Non-taxable Minimum Count. Only after the amount corrections should be pressed the button Process. If lines were processed before, but not approved, then they need to be processed again.

    In order not to change the non-taxable minimum amount each month, (because it is changed on demand by the employee), It is recommended to ask employee to insert a check mark in EDS profile.

    System is processing the additional reliefs: First, second and third group disability. For other additional reliefs (Politically Repressed Persons, or Members of the National Resistance Movement) a warning for manual processing is shown. In case the disability is up to a particular date and the employee is entitled to non-taxable minimum by EDS data, the system will check every non-taxable minimum record (Had this employee a disability for at least one day during this period) and according to that, value remains or will be canceled. Those who receives a disability allowance, are entitled also for non-applicable minimum as well as for additional relief. Those why receive pensions (for disability) are entitled for only additional relief.

    In case of manually entered data for the previous period, for example Non-applicable minimum 01.08.20 till 31.08.20, but additionally was imported data what take effect from 01.09.2020, then previously created record will not be processed. If file will includes the period Non-applicable minimum 01.08.20 till 31.12.20, then as a result for August will be 2 Non-applicable minimum entries. We recommend that either all time import or by entering information manually, leave the record open (without the end date), Then the system will cancel this record and set up correct.

    All changes can be viewed in the column Change Count. A performed action is identified for each line, what was done after the approval.

    Salary books in the list of import changes can be corrected. for example, if employer don't want to import data on dependants those entries can be sorted out and deleted. In case if the changes are correct, select the correct lines and press the button Approve. The changes will be made to the employees' payroll components. Following settings in the Social Statuses must be set if also retired employees need to be processed:

    When processing the data with retired employees, system checks the Social Status Code field in the employee card. In case if code is DP ( related to pensioners), then processing of the Non-applicable minimum is similar to ordinary employees. Social rates will be checked in any case.

    In case if prior to import, to employee will be assigned a status code DN and will be received information on retirement, the Date From of on-applicable minimum is the same as it will be on next month's first date. But, in case if DP status is al;ready in the system, then non-applicable minimum date is the same as on-applicable minimum date from EDS file.
