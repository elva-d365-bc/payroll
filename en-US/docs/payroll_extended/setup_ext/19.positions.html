    Positions and position orders

    The position order is used, to book any changes associated with the position – creating a new position, closing, load change as well as other activities with positions. When posting position order, the system automatically creates changes in the position card.

    Position order

    To multiple actions can be combined in one position order (both one and several employees at the same time), position order consists of the header and several lines. In header should be specified overall information for the whole order. A new order can be created: Position Order List - +New.

    Tab General includes following fields:

    • No. – order number (from No. Series);
    • Posting Date – date, when this order come in force;
    • Status – order status (Created, In Approval, Completed).

    In the tab Signatures It is possible to choose two signatories from the company officials register.

    In the tab Add.info specifies information about when the order has been created and who did it.

    In the tab Templates choose header templates code and footer template code.

    Each line contains the following information about one position:

    • Exist. Position Code – position code, what should be changed;
    • New Position Code – new position;
    • Description – name of the new position;
    • Date Created;
    • Date Closed;
    • Profession Code – from the list Employee Statistics Groups;
    • Department code – from the list Departments;
    • Total Loading – total loads that are intended for this position;
    • Salary Amount;
    • Vacation Days;
    • Add. Vacation Days;
    • Line Template Code – for each order line can be defined a different document template, what appears as separate order paragraph in the whole order. Creating the document templates is described in the chapter Creating Documents.

    Order on changes in existing position

    In order to make any changes to the already existing position, position orders are used. By choosing Existing Position Code, the system automatically completes all fields from position cards relating to this position. In the position order makes the necessary adjustments.

    If position should be closed and replaced with another, Then a new position should be set up at first. First you need to create a new Personal order to replace existing position with new one (described in chapter Personal Orders and only then existing position can be closed. If the user will closes the existing posts first of all, when they still be attached to employees, the system will then issue a warning and will not allow to post an order.

    All fixes can be made when the order is in status Created. If the corrections are made to profession code, then these changes also be displayed in the employee card, only when in the positions' card will be started the function Process - Update Employee Position. In update operations enable Update Profession Codes.

    Approving and posting the position orders

    When all the information you need is entered, then click Document - Create Document and create a position order printout with the attached document template. When the system has been created a document, then open the document in MS Word file by clicking on the button Open Document.

    Creating and attaching the document templates is described in the chapter Creating Documents. Print the prepared document and forward for signatures, change the status to In Approval. Upon receipt of a signed document, the status of the order can be changed to Completed and order can be posted. Order status is manually changed depending on the document condition. The document content change is not allowed for documents in status In Approval or Completed. Only completed document can be posted.

    After posting the order, it is moved to the posted order archive list with assigned the special posted order numbering series, and can not be corrected.

    To ensure that the numbers of the posted document are sequentially, the initial document number is used as a temporary number – only until the document is not posted. At the posting moment, next available numbering for posted invoices is used and applied to posted position order. The initial document number at the posted document is also saved. The number settings and definition is described in the chapter Personnel Setup.
